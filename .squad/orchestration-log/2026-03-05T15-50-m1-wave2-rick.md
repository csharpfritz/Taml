### Phase 2: Specification Synchronization

**New Sections Added:**

1. **Section 7.6 — Raw Text Blocks** (7 requirements)
   - Syntax: `...` indicator
   - Indentation handling (content preserves interior indent, strips parent level)
   - Content preservation (tabs, newlines, blank lines)
   - Termination rules (same/lesser indentation)
   - Edge cases (empty raw text, nested raw text)

2. **Section 7.7 — Data Structure Types** (3 detection requirements)
   - Maps: Key-value pairs, unordered
   - Collections: Indexed lists of objects
   - Structure detection rules (how parser determines which type)

3. **Section 7.4.3 — Collection of Objects** (4 requirements)
   - Duplicate bare keys mechanism
   - Creation rules (duplicate keys at same level → list)
   - Representation (list of dicts)
   - Round-trip serialization

4. **Section 8.2.3 — ISO 8601 Date/Time Detection** (4 requirements)
   - Supported patterns: date-only (YYYY-MM-DD), datetime (with time), timezone support
   - Timezone forms: Z (UTC), ±HH:MM offsets
   - Week dates, ordinal dates, durations, intervals (listed for future extension)
   - Type conversion requirements

5. **Section 9.4 — Raw Text Validation** (2 rules)
   - Raw text indicator must be `...` alone on value
   - Content lines must have greater indentation than parent

6. **Section 12.5 — Raw Text Security** (guidelines)
   - Injection risk considerations for embedded content
   - Size limit recommendations
   - Safe handling of untrusted input

**Updated Sections:**

- **Section 8.2.2 (Type Coercion):** Extended with 6 boolean patterns, type detection precedence order, explicit 1/0 integer rule
- **Section 6.4.1 (Key Token):** Key character rules — any Unicode except tab/newline
- **Section 6.4.2 (Value Token):** Trailing whitespace trimming standardized
- **Section 7.3.1 (Parent Node):** Empty parents → empty maps rule
- **Section 8.3 (Type System):** Expanded from 5 to 11 types with 6 inference rules
- **Section 10.1 (Error Categories):** 3 new error codes for raw text validation
- **Section 11 (Conformance):** Updated parser (12 MUST) and serializer (12 MUST) requirements
- **Appendix A (EBNF Grammar):** Fully rewritten with raw text, boolean, date, structure productions (10 grammar notes)
